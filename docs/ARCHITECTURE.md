# 架构说明

## 数据流

1. 浏览器将自然语言发送至 `/strategy/parse`。
2. 合规层先拦截红线请求；其余输入依次尝试 DeepSeek、Kimi，模型异常或无密钥时使用规则解析器。所有路径只允许返回 `StrategySpecV1`。
3. Pydantic 对指标、操作符、日期和风险参数执行白名单校验。
4. 代码预览由模板生成，不执行 LLM 原始代码。
5. 后台任务读取本地缓存行情并运行 T+1 日级回测。
6. API 返回真实计算所得指标、曲线、交易记录和数据来源。

## 开发与公测模式

- 本地开发模式：无密钥时可使用固定种子的确定性市场序列，便于离线开发与自动化测试。
- 开放公测模式：仅使用经过版本化、完整性校验和授权确认的真实行情；数据服务异常时失败关闭，不得静默使用演示行情。
- 数据层按市场适配器拆分中国 A 股、港股和美股，统一交易日历、币种、时区、复权、公司行动和证券标识。
- 每次回测保存策略哈希、数据快照版本、引擎版本、费用模型和汇率版本，保证结果可复现。
- 数据来源、更新时间、币种和延迟状态必须在结果界面显示。

## 产品化目标架构

当前实现仍是单工作区、沪深 300 代理行情和进程内后台任务，只能视为产品化迁移起点。开放公测前需要完成：

1. 账号、会话、工作区隔离和基于角色的权限控制。
2. PostgreSQL 正式迁移、持久任务队列、对象存储与幂等任务恢复。
3. 沪深 300 成分股、港股和美股的历史行情、财务快照、交易日历、复权与公司行动处理。
4. 统一订单与交易网关；模拟盘先行，实盘适配器默认关闭并受独立权限和风控策略保护。
5. 不可篡改的策略、回测、订单、权限和管理操作审计日志。
6. 指标、日志、追踪、告警、备份恢复、限流和密钥管理。

完整阶段和发布门槛见 [产品化计划](PRODUCTIZATION_PLAN.md)。
