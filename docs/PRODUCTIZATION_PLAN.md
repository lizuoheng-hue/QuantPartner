# QuantPartner 开放公测产品化计划

## 实施状态（2026-06-21）

- 已完成：账号注册登录、可撤销会话、工作区数据隔离、角色字段、审计事件、Alembic 基线、Redis 持久任务队列 worker、多市场策略契约、模拟订单生命周期、公测登录与模拟盘界面。
- 已完成：桌面和移动端“注册—策略—回测—模拟下单/撤单”浏览器验收。
- 待接入：具备再分发授权的港美股真实数据源，以及具体实盘券商适配器和凭据。
- 待完成：邮箱验证/密码重置、工作区成员管理、细粒度 RBAC、实时行情与持仓对账、限流告警、P1 分析和报告能力。

## 修订后的产品假设

- 交付目标是开放公测，路演已经完成。
- 本地和迁移期保留单一演示工作区；公测用户数据必须按账号和工作区隔离。
- 首批市场覆盖沪深 300、港股和美股，历史区间为 2019 年至最新可用交易日。
- 公测需要交易能力：模拟盘作为首个公开能力；实盘通过统一交易网关接入，但默认关闭，须通过实名、适当性、风控、审计和合规验收后按白名单开放。
- 产品化范围包括账号、全市场数据、持久任务队列、权限、审计、监控以及原规划 P1 功能。

## 阶段一：公测基础设施

- 账号登录、邮箱或手机号验证、密码重置、会话撤销。
- 用户、工作区、成员、角色、策略、版本、回测和订单的数据隔离。
- Alembic 数据库迁移；Redis + Celery/Arq 持久任务队列；任务重试、取消、超时和幂等。
- API 限流、配额、结构化错误、密钥管理和敏感信息脱敏。
- 指标、日志、链路追踪、异常聚合、告警与健康检查。

## 阶段二：多市场数据与回测

- 统一证券主数据，覆盖 A 股、港股和美股的代码映射、币种、时区与交易状态。
- 接入具备公测再分发授权的数据源，保存原始层、清洗层和版本化快照。
- 实现各市场交易日历、复权、分红拆股、停牌、涨跌停和退市处理。
- 支持多币种资金与基准，记录历史汇率版本。
- 用无未来数据泄漏的固定样本回归三个市场，并为费用、税费和撮合规则分别建模。

## 阶段三：模拟盘与实盘网关

- 建立统一 Broker Adapter、订单状态机、持仓、成交、资金和对账模型。
- 先上线模拟盘：实时或延迟行情、交易时段校验、拒单原因、撤单和每日对账。
- 实盘接入采用显式开关、用户二次确认、最小权限密钥、风控限额和紧急停止机制。
- 所有下单必须由用户确认；禁止 LLM 直接生成并提交订单。
- 完整记录策略版本、信号、确认人、订单请求、券商响应和状态变更。

## 阶段四：P1 产品能力

- 全市场筛选、行业轮动归因和多周期压力测试。
- Excel/PDF 报告导出、参数绩效差异与受控优化建议。
- 用户通知、配额管理、反馈闭环、运营后台和数据删除/导出能力。
- 先不开放支付、社区和无人值守自动实盘；单独立项并完成安全与合规评审。

## 开放公测发布门槛

- 不存在跨账号或跨工作区数据访问；权限测试和安全测试通过。
- 三个市场的数据完整性、复权、日历和费用模型通过基准样本校验。
- 回测和交易任务可恢复、可取消、幂等，服务重启不丢任务。
- 模拟盘完成订单全生命周期和每日对账；实盘默认关闭。
- 核心接口具备限流、审计、监控、告警和灾难恢复演练记录。
- 隐私政策、用户协议、风险揭示、数据授权和交易合规材料完成审查。
- 公测环境不使用演示行情，不在日志中保存密钥、完整自然语言输入或券商凭据。

## 关键决策

“无正式注册、单一演示工作区”与“开放公测”无法同时作为最终状态。项目采用迁移策略：现有工作区继续服务本地开发；账号与工作区隔离完成后才允许外部用户进入公测。模拟盘可以先发布，实盘能力只有在技术与合规门槛同时满足后才启用。
